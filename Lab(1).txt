1. Створіть каталог із назвою “repo”.

2. Ініціалізуйте у ньому порожній репозиторій, використовуючи інструменти командного рядка.

3. Налаштуйте перегляд каталогу таким чином, щоб було видно приховані файли та перегляньте вміст каталога “.git”.

Результат: Прикріпіть скріншот на якому видно вміст каталогу “.git” та окремо опишіть призначення кожного із елементів каталогу.



config - цей файл містить настройки для вашого сховища, тут, наприклад, зберігається url вашого сховища в сховище, ваше ім'я, email і так далі. Кожен раз коли ви робите git config, ви звертаєтеся до цього файлу.
description - використовується gitweb для пояснення сховища.
hooks - ця папка містить скрипти, які можуть виконуватися на різних етапах виконання Git. Ці скрипти, звані хукамі, можуть запускатися до / після commit / rebase / pull ... Ім'я скрипта визначає час його виконання. Прикладом хука може служити скрипт перевірки стилю перед виконанням команди push в репозиторій.
info - exclude - тут описуються файли, які ви не хочете включати в репозиторій. Функціонал цього файлу такий же, як у файлу .gitignore, за винятком того, що він не передається в репозиторій. На практиці зазвичай для всіх завдань вистачає .gitignore.
Branch, tags, HEAD - це одне і те ж

Тепер ви розумієте, що все в Git може бути отримано через правильний хеш. Давайте тепер подивимося на HEAD. Отже, що ж там?

cat HEAD
ref: refs/heads/master

Там немає хешу, і в цьому є сенс, тому що HEAD - це покажчик на верхівку гілки, з якої ви працюєте. Якщо ви подивіться в файл refs / heads / master, то побачите:

cat refs/heads/master
4cf44f1e3fe4fb7f8aa42138c324f63f5ac85828
